Update form Order Jasa Tugas jadi multi-step. Step pertama berisi Nama dan No WhatsApp, lalu setelah klik “Selanjutnya” lakukan pengecekan ke Google Sheets berdasarkan nomor WA; jika nomor sudah pernah dipakai tetapi nama berbeda, tampilkan peringatan agar menggunakan nama sebelumnya. Step kedua berisi detail tugas: jenis tugas (Makalah, PPT, Artikel, dan tambah “Tugas Harian” di `z.enum` schema Zod serta di dropdown UI `SelectItem`), jumlah halaman/slide dengan aturan berbeda per jenis (Makalah/Artikel dropdown 10–80 kelipatan 5, PPT 5–20 bisa input spesifik, Tugas Harian 2–15 bisa input spesifik), catatan maksimal 100 karakter, serta upload file JPG/PDF yang disimpan ke Google Drive (link disimpan di Google Sheets). Pastikan juga backend/API `createOrder` ikut menambahkan “Tugas Harian” di validasi enum atau switch-case agar tidak error, serta update database jika memakai enum dan logic pricing/estimasi jika ada perhitungan berdasarkan jenis tugas. Step ketiga menampilkan ringkasan order, estimasi selesai (default 3 hari, PPT 4 hari, disesuaikan dengan tipe order: super ekspres -2 hari + biaya, ekspres -1 hari + biaya, standar tanpa tambahan), serta perhitungan harga otomatis (Makalah/Artikel berdasarkan range halaman, PPT per slide, Tugas Harian per lembar). Step berikutnya adalah pembuatan order ID dan status awal “verifikasi tugas” dengan pembayaran awal (DP) 10.000 dan QRIS statis, disertai upload bukti transfer wajib sebelum lanjut ke verifikasi pembayaran. Setelah diverifikasi, status menjadi “proses pengerjaan”, lalu admin dapat upload hasil ke Google Drive namun file belum bisa diunduh sebelum pelunasan. Pada tahap pelunasan, tampil QRIS dengan sisa pembayaran, user upload bukti transfer, status kembali verifikasi pembayaran, dan setelah valid file bisa diunduh serta status menjadi selesai. Tambahkan fitur revisi maksimal 1 kali dengan upload maksimal 3 file JPG saja dan catatan maksimal 100 karakter, status berubah menjadi “revisi” selama 1x24 jam, setelah selesai kembali ke “selesai”. Semua status, file link, dan data disimpan di Google Sheets sebagai database utama, termasuk tracking payment, revision, dan status order untuk dashboard admin.
